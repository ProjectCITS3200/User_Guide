Sediment Core Analysis Pipeline

Introduction

Due to the limited number of available image samples, it is infeasible to build a robust machine learning model for sediment core detection and classification. Instead, we implemented a traditional computer vision pipeline comprising three main steps:

  1. Extraction: extracting the sediment core from the uploaded image.
  2. Scaling: determining the true length of the sediment core.
  3. Lamination analysis: calculating the colour and depth of each sediment core lamination.

Background: Digital Images

A digital colour image is represented as a 3D matrix, where each pixel’s colour is encoded by the combination of three colour channels (e.g. RGB). For instance, a  RGB image may be represented as a matrix where each element corresponds to a pixel’s red, green, and blue values (Figure 2).

Figure 2: Representation of a  RGB image composed of red, green and blue values

This representation enables straightforward image filtering, segmentation, and classification, allowing for simple conversion between colour spaces and the identification of pixel-specific properties.

Step 1: Extraction

The process of automatically locating sediment cores within images involves three stages: colour filtering, contour detection, and contour classification.

Colour filtering

The sediment core dataset consists of images with sediment cores photographed against a white background with grey shadows. To isolate the sediment core, we filter out grey and white pixels.

To achieve this, we first convert the image from BGR to HSV using OpenCV’s cvtColor method. We then define upper and lower HSV thresholds for both grey and white. For grey, the HSV range is defined as [0, 0, 50] to [180, 50, 200], while for white it is [0, 0, 200] to [180, 20, 255]. Pixels within these thresholds are filtered out, leaving the sediment core as the remaining foreground.

Contour detection

Contour detection identifies the borders of objects in an image, which are typically characterised by uniform pixel colour and intensity. Given that sediment core laminations form distinct borders, this approach allows us to detect the entire core’s border.

We preprocess the image by converting it to greyscale and applying Gaussian blur to reduce noise. Next, we apply Otsu’s thresholding to binarize the image. Using OpenCV’s findContours function, we retrieve a list of contours representing potential objects in the image.

Contour classification

Since various objects may be present in the image (e.g. hands, desks, paper, etc.), we must classify which contour corresponds to the sediment core. Sediment cores are typically large, long, thin rectangular objects located in the image foreground.

We filter out contours with an area less than 100,000 pixels, as these are likely not sediment cores. Furthermore, we discard contours where the longest side is less than twice the length of the shortest side. The largest remaining contour is then classified as the sediment core.

Step 2: Scaling

The sediment core dataset we received was corrected for lens distortion, ensuring a consistent scale across all images.

Automatically determining the true length of a sediment core by detecting a ruler within the image proved challenging. Hence, we opted for a scaling approach where the core’s width is measured manually. The user provides the real-world width of the sediment core in millimetres, and we calculate the length based on the pixel dimensions of the core.

The image dataset we received consisted of sediment core images that had already been corrected for lens distortion. Thus, we can assume that the scale of the sediment core is consistent throughout the entire image.

Step 3: Determining Lamination Colours

To analyse colour variations across the sediment core, we calculate the average weighted colour for each pixel-wide segment across the core’s width. This allows us to capture the colour changes across the laminations with maximum precision.

Retrieving Weights

As colours near the centre of the core are more representative of the lamination’s true colour (compared to the edges, which may include shadows or artefacts from the sediment core), we use a weighted average to calculate the lamination colour. Weights increase linearly towards the centre and decrease symmetrically toward the edges. For example, a core with a width of five pixels would have the weights [0, 1, 2, 1, 0].

Application Functionality Guide

Below are instructions and an explanation of what functionality the application has, and where to find all of these functions for use.

General Layout

Toolbar

This menu, at the top of the application is for loading an image from your computer into the application, for altering the colour space of the image’s colour data, and for accessing helpful resources in navigating the application (like this document).

Analysis Space

The analysis space sits below the toolbar. Upon opening the application the analysis space will contain two empty white panels. While running the application these two panels will be either Image Panels or Graph Panels. Image Panels display an image of the sediment core being analysed. A Graph Panel shows the output of application’s analysis with two graphs known as the Layers plot and the Colours plot.

Image Thumbnail

The image thumbnail sits beneath the Analysis Space and is at the very bottom of the application. The thumbnail displays images that the user loads into the application. It arranges the loaded images horizontally across the bottom of the application. This allows all loaded images to be displayed in the application.

The user analyses an image by selecting it from the thumbnail. Selecting an image opens a box which prompts the user to choose one of the three options.

  1. Left Graph: This displays a Graph Panel in the Left Panel of the Analysis Space.
  2. Right Graph: This displays a Graph Panel in the Right Panel of the Analysis Space.
  3. Image and Graph: This displays an Image Panel in the Left Panel and a Graph panel in the Right Panel.

If the panel the user selected (Left Graph or Right Graph) already has a Graph or Image Panel (from a previous image) displayed on it when choosing to display a new image, the old Graph Panel is deleted and a new Graph Panel is rendered. Essentially, the application can show Graph Panels for two images and upon user request can change 1 Graph Panel to show a new Graph Panel for a new image. Selection of the Image and Graph option will always reload both panels of the analysis space.

Image Panel

The image panel displays an image of the core being analysed. It has its own toolbar with 2 buttons. These buttons are:

  1. Flip/Invert Button: This button flips the direction that the sediment core is analysed in. This results in the bottom of the core being on the top of the Graph Panel or the bottom of the core being on the bottom of the Graph Panel.

  1. Image Calibration Button: This button opens a box which prompts the user to input the width of the sediment core (in mm). The default setting is 76mm. Specifying the width improves the reliability of analysis by making it easier for the application to measure the length of the sediment core. After inputting a number representing the core’s width, the graphing window quickly reappears with updated measurement values corresponding to your input number. Trying to press this button without an image loaded simply does nothing.

Graph Panel

These panels contain two boxes each displaying a visualisation of the application’s analysis. The graph on the left, known as the Layer Graph, shows a layered colour recreation of the core being analysed. The graph on the right, known as the Colours Graph, plots the colour values along the length of the sediment core. The Colours Graph and the Layers Graph share the same y-axes. This means a layer on the y-axes of either graph corresponds to the same slice of data from the core. The graphs panel, like the image panel, has its own toolbar. The tool bar allows the user to interact with the plots. The toolbar has the following 9 buttons.

  1. Revert Button: Redraw the Colours Graph and Layers Graph in its original form.  
  2. Back Button: Undo the previous change to the Colours Graph.  
  3. Forward Button: Redraw a previous version of the Colours Graph after using the Back Button.
  4. View Adjust Button: Drag the plotted data across the Colours Graph Axes to get a new view of the data.
  5. Zoom-in Button: Zoom in by drawing a box on the Colours Graph which is then rendered across the Colours Graph Axes. .
  6. Change Units Button: Switch between plotting raw colour values or intensity (%) colour values.
  7. Grid Lines Button: Toggle on and off gridlines on the Colours Graph.
  8. Export as CSV: Export the data from the Colours Graph as a comma separated value file.
  9. Export as Picture: Save an image of the ColourGraph.

Toolbar

As described in the General Layout Section the toolbar sits at the top of the application and has a series of buttons that initiate the analysis of the sediment core. These buttons are:

  1. Open: This button opens the computers directory and allows the user to select images of sediment cores to load into the application for analysis.
  2. Help > User Guide: This button opens a link to this User Guide.
  3. Help > About: This button takes the user to the GitHub repository.
  4. Run RGB Analysis Button : This button re-renders currently loaded Graph Panels to show the Colours Plot with Red, Green and Blue values (RGB) against the core’s width.
  5. Run CIELAB Analysis Button: This button re-renders currently loaded Graph Panels to show the Colours Plot with L*, a*, and b* values (LAB) against the core's width.
